## 一、学生群体

### 1.1 需要的写作工具类型

| 类型 | 具体场景 | 优先级 |
|------|----------|--------|
| **作文写作** | 记叙文、议论文、说明文、散文、读后感 | 高 |
| **论文写作** | 课程论文、毕业论文、文献综述、开题报告 | 高 |
| **实验报告** | 物理/化学/生物实验报告、工程实验报告 | 中 |
| **学习笔记** | 课堂笔记整理、知识点总结、思维导图 | 中 |
| **PPT/汇报** | 课程汇报、答辩PPT、小组展示 | 中 |
| **申请文书** | 奖学金申请、入党申请、社团申请 | 低 |
| **英语写作** | 英语作文、托福/雅思写作、翻译练习 | 中 |

### 1.2 特殊需求

#### A. 学习辅助需求
- **大纲生成**: 根据题目生成写作大纲和思路
- **素材推荐**: 提供名人名言、案例、论据素材
- **范文参考**: 提供优秀范文供学习参考
- **写作指导**: 分步骤引导完成写作，而非直接给出答案

#### B. 格式规范需求
- **论文格式**: 自动生成符合学校要求的格式（字体、行距、页边距）
- **引用格式**: 支持GB/T 7714、APA、MLA等引用格式
- **目录生成**: 自动生成论文目录
- **参考文献**: 自动整理和格式化参考文献

#### C. 学术能力提升
- **语法检查**: 检查语法错误和表达不当
- **逻辑分析**: 分析文章逻辑结构，提出改进建议
- **查重预检**: 提供初步的相似度检测
- **润色优化**: 提升学术表达水平

#### D. 多语言需求
- **中英互译**: 学术文献翻译
- **英语写作辅助**: 英语作文批改和润色
- **摘要翻译**: 中英文摘要互译

### 1.3 重要注意事项

#### 学术诚信问题（核心关注点）

| 风险 | 说明 | 应对措施 |
|------|------|----------|
| **抄袭风险** | 直接使用AI生成内容可能被判定抄袭 | 提供"写作辅助模式"而非"代写模式" |
| **查重风险** | AI生成内容可能被AI检测工具识别 | 提供改写、降重功能，但需警示用户 |
| **学校政策** | 不同学校对AI工具使用政策不同 | 明确提示用户遵守学校规定 |
| **能力培养** | 过度依赖AI影响写作能力提升 | 引导式辅助，而非直接生成 |

#### 建议的产品策略

1. **定位为"学习助手"而非"代写工具"**
   - 提供思路引导、大纲建议
   - 给出修改建议而非直接改写
   - 提供范文学习而非直接套用

2. **添加学术诚信提醒**
   - 首次使用显示学术诚信提示
   - 生成内容附带"仅供参考"声明
   - 引导用户理解和消化AI建议

3. **分级功能设计**
   - 基础版：大纲、素材、指导
   - 进阶版：润色、优化（需勾选已理解学术诚信）

### 1.4 功能建议清单

│   ├── 大纲规划

│   ├── 引用格式化
│   ├── 查重预检
│   └── 摘要生成
│
├── 实验报告
│   ├── 报告模板

## 二、教师群体

### 2.1 需要的写作工具类型

| 类型 | 具体场景 | 优先级 |
|------|----------|--------|
| **教案设计** | 单元教案、课时教案、教学设计 | 高 |
| **课件制作** | PPT课件、教学材料 | 高 |
| **学生评语** | 期末评语、作业评语、成绩单评语 | 高 |
| **试卷出题** | 选择题、填空题、简答题、作文题 | 中 |
| **教学反思** | 课后反思、教学总结 | 中 |
| **通知公告** | 班级通知、家长通知、活动通知 | 中 |
| **工作总结** | 学期总结、年度总结、述职报告 | 中 |
| **论文写作** | 教学研究论文、课题申报 | 低 |

### 2.2 特殊需求

#### A. 教案设计需求
- **课标对齐**: 自动对齐新课标要求
- **学情适配**: 根据年级、学科自动调整难度
- **环节完整**: 导入、新授、练习、小结、作业
- **时间分配**: 自动建议各环节时间
- **教学方法**: 推荐适合的教学方法

#### B. 评语生成需求
- **个性化**: 根据学生特点生成差异化评语
- **正向引导**: 以鼓励为主，委婉指出不足
- **多维度**: 学习、品德、体育、艺术等多维度
- **批量生成**: 支持批量生成全班评语
- **去重功能**: 确保每个学生评语不重复

#### C. 试卷出题需求
- **知识点覆盖**: 根据教学内容自动覆盖知识点
- **难度分级**: 支持简单/中等/困难分级
- **题型多样**: 选择、填空、判断、简答、计算
- **答案解析**: 自动生成标准答案和解析
- **试卷排版**: 自动生成规范的试卷格式

#### D. 效率提升需求
- **模板复用**: 保存常用模板快速复用
- **批量处理**: 批量生成同类型内容
- **快速编辑**: 简单修改即可使用
- **历史记录**: 方便查找之前的内容

## 三、公务员/体制内群体

### 3.1 需要的写作工具类型

| 类型 | 具体场景 | 优先级 |
|------|----------|--------|
| **公文写作** | 通知、通报、报告、请示、批复、函 | 高 |
| **讲话稿** | 领导讲话、会议发言、致辞、主持词 | 高 |
| **工作报告** | 工作总结、工作计划、述职报告、调研报告 | 高 |
| **汇报材料** | 工作汇报、专题汇报、情况说明 | 高 |
| **会议纪要** | 会议记录、会议纪要整理 | 中 |
| **制度文件** | 规章制度、工作方案、实施意见 | 中 |
| **宣传材料** | 新闻稿、宣传文案、简报 | 中 |
| **党建材料** | 党课讲稿、心得体会、民主生活会材料 | 中 |

### 3.2 特殊需求

#### A. 公文格式规范（核心需求）

公文必须严格遵循《党政机关公文格式》（GB/T 9704-2012）：

| 要素 | 规范要求 |
|------|----------|
| **文号** | 发文机关代字+年份+发文顺序号 |
| **标题** | 发文机关+事由+文种 |
| **主送机关** | 规范使用全称或规范简称 |
| **正文** | 首行缩进2字符，段落明确 |
| **附件** | 附件说明规范 |
| **落款** | 发文机关署名、成文日期 |
| **字体** | 仿宋体3号字（标题除外） |
| **版式** | A4纸，上3.7cm下3.5cm左2.8cm右2.6cm |

#### B. 语言风格要求
- **严谨性**: 用词准确，逻辑严密
- **规范性**: 使用规范的公文用语
- **政治性**: 符合政策导向，政治正确
- **简洁性**: 言简意赅，不说空话套话

#### C. 领导讲话稿需求
- **场合适配**: 不同场合语言风格不同
- **层次分明**: 观点清晰，条理分明
- **金句提炼**: 适当提炼金句，增强感染力
- **时长控制**: 根据时长要求控制字数

#### D. 政策对齐需求
- **政策术语**: 使用最新的政策术语表达
- **精神对齐**: 对齐最新的会议精神、政策要求
- **数据引用**: 准确引用统计数据

### 3.4 公文用语规范库

需内置以下规范用语：

| 类别 | 示例 |
|------|------|
| **开头用语** | 为了、根据、按照、遵照、为进一步 |
| **引述用语** | 特此通知、特此函告、特此公告 |
| **承启用语** | 为此、据此、鉴于、综上所述 |
| **结尾用语** | 请审批、请批复、请遵照执行 |
| **称谓用语** | 贵单位、你局、本委、我厅 |

### 3.5 注意事项

1. **敏感内容过滤**: 涉密内容不上传，提示用户注意信息安全
2. **政治正确性**: 确保生成内容符合政策导向
3. **最新政策同步**: 定期更新政策术语和表达方式
4. **格式严格遵守**: 公文格式必须100%符合国家标准

- **多语言**: 中英文商务邮件
- **邮件优化**: 主题行优化、正文精简

#### B. 工作汇报需求
- **数据展示**: 自动生成图表、可视化

#### C. 方案策划需求
- **框架完整**: 自动生成完整方案框架
- **数据支撑**: 提供行业数据参考
- **创意建议**: 提供创意方向建议
- **预算规划**: 辅助预算编制

#### D. 效率需求
- **快速生成**: 输入关键信息即可生成
- **一键复制**: 方便粘贴到其他工具
- **多格式导出**: Word、PPT、PDF
- **团队协作**: 模板共享、协作编辑

## 六、总结

### 核心发现

1. **学生群体**：需要"学习辅助"而非"代写"，学术诚信是核心关注点
2. **教师群体**：教案和评语是刚需，批量处理能力很重要
3. **公务员群体**：公文格式规范是第一要求，政治正确性必须保证
4. **企业群体**：效率为先，场景覆盖要全，多语言商务需求强

### 差异化竞争建议

1. **格式规范**：公文格式、论文格式做到100%规范
2. **场景深耕**：每个群体做5-10个核心场景做精做透
3. **批量能力**：教师评语、企业邮件等批量需求要满足
4. **合规安全**：学术诚信提醒、敏感内容过滤

### 技术实现建议

1. 建立各领域专业术语库和表达库
2. 针对不同群体训练/微调专属模型
3. 完善的模板系统支持快速定制
4. 严格的内容安全和合规检测机制
