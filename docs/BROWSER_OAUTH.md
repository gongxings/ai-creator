# Browser OAuth 授权流程

本文档介绍如何在 ai-creator 项目中使用浏览器进行 OAuth 授权。

## 概述

Browser OAuth 是一种通过真实浏览器窗口完成授权的方式：
- 用户在前端页面看到真实的浏览器窗口
- 手动登录目标平台账号
- 系统自动保存登录凭证（Cookie）
- 无需手动复制粘贴 Cookie

这种方式的优势：
1. **避免风控**：使用真实浏览器，不容易被平台检测
2. **自动保存**：登录成功后自动获取和保存所有必要凭证
3. **用户体验好**：可视化操作，直观清晰

## 前置要求

### 服务启动

需要同时启动后端和前端服务：

**终端1 - 启动后端：**
```bash
cd E:\study\opensource\ai-creator-new\backend
python run.py
```

**终端2 - 启动前端：**
```bash
cd E:\study\opensource\ai-creator-new\frontend
npm run dev
```

### 端口配置

| 服务 | 默认端口 | 说明 |
|------|---------|------|
| 前端 | 3000 | Vite 开发服务器 |
| 后端 | 8001 | FastAPI 服务 |

确保后端端口为 **8001**（可在 `.env` 文件中配置 `PORT=8001`）。

## 授权流程

### 步骤 1：登录系统

1. 打开浏览器访问：http://localhost:3000
2. 使用已有账号登录，或先注册新账号

### 步骤 2：进入 OAuth 授权页面

1. 点击右上角 **"设置"** 链接
2. 在左侧菜单选择 **"OAuth账号"** 标签
3. 点击 **"添加账号"** 按钮

### 步骤 3：选择授权方式

在弹出的对话框中：

| 字段 | 说明 |
|------|------|
| 选择平台 | 选择要授权的平台（如：豆包） |
| 账号名称 | 为这个授权起个名字（可选） |
| Auth Mode | 选择 **"Browser"** 模式 |

选择完成后，点击 **"打开浏览器授权"** 按钮。

### 步骤 4：浏览器授权

系统会自动跳转到 Playwright 授权页面，显示内容如下：

```
┌─────────────────────────────────────────┐
│  豆包 Playwright 授权                   │
│  ┌─────────────────────────────────┐    │
│  │                                 │    │
│  │    [浏览器实时画面区域]          │    │
│  │                                 │    │
│  │                                 │    │
│  └─────────────────────────────────┘    │
│                                         │
│  授权说明：                              │
│  1. 请在下方浏览器中登录 豆包 账号       │
│  2. 登录成功后，系统会自动保存凭证       │
│  3. 凭证保存后会自动返回设置页面        │
│                                         │
│  [开始流] [截图] [刷新] [断开]          │
└─────────────────────────────────────────┘
```

**操作：**
1. 在浏览器窗口中操作
2. 找到豆包的登录入口
3. 使用手机扫码或账号密码登录
4. 登录成功后等待系统保存凭证

### 步骤 5：授权完成

当看到以下提示时，表示授权成功：

```
✅ 授权完成！账号: doubao
```

系统会在 **3秒后** 自动返回设置页面。

## 授权后检查

返回设置页面后，在 OAuth 账号列表中应该能看到新添加的账号：

| 平台 | 账号名称 | 状态 |
|------|---------|------|
| 豆包 | doubao | 正常 |

## 常见问题

### Q1: WebSocket 连接失败

**错误信息：** `WebSocket未连接`

**解决方法：**
1. 确保后端服务正在运行（端口8001）
2. 确保前端通过 `npm run dev` 启动（使用Vite代理）
3. 刷新页面重试

### Q2: 看不到浏览器画面

**可能原因：**
1. Playwright 未安装
2. 浏览器被系统阻止

**解决方法：**
```bash
# 安装 Playwright 浏览器
pip install playwright
playwright install chromium
```

### Q3: 登录后提示"凭证保存失败"

**可能原因：**
1. Cookie 获取失败
2. 网络连接问题

**解决方法：**
1. 重新操作登录流程
2. 确保网络连接正常
3. 等待页面完全加载后再操作

### Q4: 生成的图片无法访问

**可能原因：**
1. Cookie 已过期
2. 凭证不完整

**解决方法：**
1. 重新进行授权流程
2. 确保登录后有足够的使用时间

## 技术实现

### 前端组件

- **PlaywrightViewer.vue**: 实时浏览器画面显示
- **WebSocket 连接**: 实时推送浏览器画面和日志
- **UserSettings.vue**: 授权对话框和账号管理

### 后端服务

- **websocket.py**: WebSocket 端点处理
- **playwright_service.py**: Playwright 自动化控制
- **oauth_service.py**: OAuth 凭证保存和管理

### WebSocket 消息类型

| 消息类型 | 说明 |
|---------|------|
| `screenshot` | 浏览器截图 |
| `log` | 操作日志 |
| `url` | 当前页面URL |
| `credentials_saved` | 凭证保存完成 |
| `error` | 错误信息 |

## 注意事项

1. **Cookie 有效期**：平台 Cookie 可能会过期，需要定期重新授权
2. **频率限制**：部分平台有 API 调用频率限制
3. **多账号支持**：可以授权多个账号，系统会分开管理
4. **安全性**：凭证会加密存储在数据库中
